Skills are domain knowledge modules that agents invoke automatically during workflow execution. Each skill is a `SKILL.md` file in `.github/skills/` containing patterns, decision heuristics, IaC samples, and cross-references for a specific problem domain.

## How Skills Are Loaded

Agents auto-discover skills via the Copilot runtime when they encounter a domain that requires specialized knowledge. Users do not invoke skills directly â€” the responsible agent selects and loads the appropriate skill based on the current workflow step and problem context.

The skill body follows a standard structure:

1. **YAML frontmatter** â€” metadata including `name`, `description`, `category`, and `used_by`
2. **Overview** â€” scope, CAF/WAF alignment, when to use vs. not use
3. **Patterns & Guidance** â€” architecture patterns, decision heuristics, IaC samples
4. **Brownfield Playbook** (service skills) â€” assessment and hardening steps
5. **Anti-Patterns** â€” common mistakes to avoid
6. **Cross-References** â€” related skills, ADRs, and Microsoft Learn links
