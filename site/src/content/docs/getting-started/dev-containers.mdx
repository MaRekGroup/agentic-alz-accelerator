The repository includes a Dev Container configuration with all tools pre-installed.

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) or compatible runtime
- [VS Code](https://code.visualstudio.com/) with the [Dev Containers extension](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers)

## Open in Dev Container

1. Clone the repository
2. Open in VS Code
3. Press `F1` → **Dev Containers: Reopen in Container**

The container includes:

| Tool | Purpose |
|------|---------|
| Azure CLI (`az`) | Azure resource management |
| Bicep CLI | Bicep template compilation and linting |
| Terraform CLI | Terraform plan/apply |
| GitHub CLI (`gh`) | Workflow triggering, PR management |
| Python 3 + pip | Validators, MCP servers, diagram generation |
| Node.js + npm | Optional validation scripts |

## Post-Setup

```bash
# Activate Python virtual environment
source .venv/bin/activate
pip install -r requirements.txt

# Authenticate Azure CLI
az login --use-device-code

# Authenticate GitHub CLI
gh auth login
```

## GitHub Copilot Agents

The Dev Container is configured with GitHub Copilot agent definitions in `.github/agents/`.
Use `@workspace` in Copilot Chat to invoke the orchestrator or individual agents.

## Local Development Guardrails

The devcontainer automatically installs [Lefthook](https://github.com/evilmartians/lefthook) via `npx --yes lefthook install` in `postCreateCommand`. Lefthook enforces quality gates at three git hook stages:

| Hook | Checks |
|------|--------|
| **pre-commit** | Security baseline validation (Bicep/Terraform), cost governance validation, `terraform fmt`, `terraform validate`, `ruff` (Python lint), `yamllint` |
| **commit-msg** | Conventional Commits format via `commitlint` |
| **pre-push** | Full security + cost scans, `pytest` |

Commit messages must follow Conventional Commits format (`feat:`, `fix:`, `docs:`, `chore:`, etc.). See `commitlint.config.js` for the full list of accepted types.

:::note[Bypass when needed]
The pre-push `pytest` job currently has pre-existing collection errors unrelated to most contributions. Use `git push --no-verify` to bypass pre-push hooks when needed. CI re-runs the same validators on every PR via `.github/workflows/5-pr-validate.yml`.
:::

### Copilot Runtime Hooks (Separate Concept)

The git hooks above are distinct from **Copilot runtime hooks** in `.github/hooks/`. Runtime hooks wrap Copilot sessions with guards (tool-guardian, secrets-scanner) and telemetry — they are not triggered by git operations.

See [`.github/hooks/README.md`](https://github.com/MaRekGroup/agentic-alz-accelerator/blob/main/.github/hooks/README.md) for the runtime hook inventory.
