Every agent and IaC module maps to an official Azure Landing Zone design area
from the [Cloud Adoption Framework](https://learn.microsoft.com/azure/cloud-adoption-framework/ready/landing-zone/design-areas).

## Design Area Mapping

| CAF Design Area | Bicep Module | Terraform Module | Agent(s) |
|-----------------|-------------|-----------------|----------|
| Billing & Tenant | `billing-and-tenant/` | — | Scribe, Oracle |
| Identity & Access | `identity/` | `identity/` | Warden, Forge |
| Resource Organization | `governance/`, `policies/` | `policies/` | Oracle, Strategist |
| Network Topology | `connectivity/`, `networking/` | `connectivity/` | Oracle, Forge |
| Security | `security/`, `platform-security/` | `platform-security/` | Warden, Sentinel |
| Management | `management/`, `logging/` | `platform-management/` | Chronicler, Sentinel |
| Governance | `governance/`, `policies/` | `policies/` | Warden, Sentinel |
| Platform Automation | CI/CD pipelines | CI/CD pipelines | Envoy, Forge |

## How Agents Use Design Areas

### Step 1 — Requirements (Scribe)
Maps user requirements to all 8 design areas, ensuring nothing is missed.

### Step 2 — Architecture (Oracle)
Assesses each design area against the Well-Architected Framework (WAF) pillars
and provides recommendations.

### Step 3.5 — Governance (Warden)
Discovers Azure Policies applicable to each design area and generates
governance constraints.

### Step 4 — Planning (Strategist)
Selects AVM modules for each design area and plans the deployment order.

### Step 5 — Code Generation (Forge)
Generates IaC templates organized by design area, with cross-references
to governance constraints.

### Step 0 — Assessment (Assessor) — Brownfield Only
For brownfield scenarios, the Assessor evaluates all 8 design areas against the
existing environment’s resources, identifying gaps and generating a migration
roadmap before the standard workflow begins. See the
[Brownfield Assessment Guide](/agentic-alz-accelerator/guides/brownfield-assessment/).

## Design Area Details

### A — Billing & Active Directory Tenant

Enterprise enrollment, management group root, tenant setup. Foundation of the entire hierarchy. The Scribe agent captures tenant and billing requirements, and Oracle validates against CAF guidance.

### B — Identity & Access Management

Entra ID, Managed Identity, RBAC assignments, PIM, conditional access policies. The Warden agent discovers existing IAM policies and Forge generates compliant identity resources.

### C — Resource Organization

Management group hierarchy, subscription vending, resource groups, naming conventions, tagging. Oracle maps the organization structure and Strategist plans resource group layout following CAF naming.

### D — Network Topology & Connectivity

Hub-spoke, VNet, NSG, DDoS protection, Azure Firewall, Private DNS, ExpressRoute/VPN. Oracle designs the topology and Forge generates the networking modules.

### E — Security

Defender for Cloud, Microsoft Sentinel, Key Vault, security baseline enforcement, SOAR playbooks. Warden enforces security policies and Sentinel monitors security posture continuously.

### F — Management

Log Analytics, Azure Monitor, Automation Account, Update Management, Backup, Action Groups. Chronicler documents the management configuration and Sentinel uses it for ongoing monitoring.

### G — Governance

Azure Policy (custom + built-in), budget alerts, Cost Management, compliance reporting, RBAC governance. Warden discovers policies and Sentinel scans for compliance continuously.

### H — Platform Automation & DevOps

CI/CD pipelines, GitOps, IaC validation, deployment automation, agent-driven operations. Envoy manages deployments and Forge generates the IaC that pipelines consume.
