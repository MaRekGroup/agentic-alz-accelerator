Platform Landing Zones are the shared foundation that all Application Landing
Zones depend on. They must be deployed before any workload subscription.

## Deployment Order

```
[1] Management    вҶҗ Deploy first (all other LZs log here)
       |
[2] Connectivity  вҶҗ Deploy second (all workloads network here)
       |
[3] Identity      вҶҗ Deploy third (workloads that need domain join)
       |
[4] Security      вҶҗ Deploy fourth (SecOps, Sentinel, Defender)
       |
[5+] App LZs      вҶҗ corp, online, sandbox, sap ...
```

## Management (`platform-management`)

Centralized observability and operations for the entire Azure estate.

| Resource | Details |
|---------|---------|
| Log Analytics Workspace | PerGB2018, 365-day retention, central for all subs |
| Automation Account | Linked to Log Analytics; start/stop runbooks |
| Budget | 80/100/120% forecast alerts |

## Connectivity (`platform-connectivity`)

Hub networking with selectable topology (Hub-Spoke or Virtual WAN).

| Resource | Details |
|---------|---------|
| Hub Virtual Network | Configurable address space and subnets |
| Azure Bastion | Secure RDP/SSH without public IPs |
| Azure Firewall | Optional вҖ” Premium SKU for TLS inspection |
| Budget | 80/100/120% forecast alerts |
