Platform Landing Zones are the shared foundation that all Application Landing
Zones depend on. They must be deployed **in strict order** before any workload subscription.

## Deployment Order

```
[1] Management    вҶҗ Deploy first (all other LZs log here)
       в”Ӯ
[2] Connectivity  вҶҗ Deploy second (all workloads network here)
       в”Ӯ
[3] Identity      вҶҗ Deploy third (workloads that need domain join)
       в”Ӯ
[4] Security      вҶҗ Deploy fourth (SecOps, Sentinel, Defender)
       в”Ӯ
[5+] App LZs      вҶҗ corp, online, sandbox, sap ...
```

:::tip[Why this order?]
Management deploys the central Log Analytics workspace that every other subscription forwards diagnostics to. Connectivity creates the hub network that spokes peer into. Identity extends AD DS into Azure (requires network). Security stands up Sentinel and Defender (requires logging + network).
:::

## Platform Landing Zone Profiles

### Management (`platform-management`)

Centralized observability and operations for the entire Azure estate.
Deploy this **first** вҖ” every other landing zone sends logs here.

| Attribute | Value |
|-----------|-------|
| Deploy Order | **1** вҖ” deploy first |
| Management Group | Platform |
| Subscription Purpose | Centralized logging, monitoring, automation |
| Key Resources | Log Analytics (365-day retention), Microsoft Sentinel, Automation Account, Recovery Services Vault |
| Policy Initiatives | Azure Security Benchmark, NIST SP 800-53, ISO 27001 |
| Microsoft Sentinel | вң… Centralized workspace |
| Budget | 80 / 100 / 120 % forecast alerts |
| Profile | `platform-management` |

:::note
The Log Analytics workspace uses `PerGB2018` pricing tier. Retention is 365 days in production and 90 days in dev. The Automation Account is linked to the workspace for Update Management and Change Tracking.
:::

### Connectivity (`platform-connectivity`)

Hub networking with selectable topology вҖ” **Hub-Spoke** (default) or **Azure Virtual WAN**.

| Attribute | Value |
|-----------|-------|
| Deploy Order | **2** вҖ” after management |
| Management Group | Platform |
| Hub Topology | Hub-spoke (default) or Azure Virtual WAN вҖ” selectable |
| Key Resources | Azure Firewall Premium, DDoS Protection Plan, 20+ Private DNS Zones, ExpressRoute/VPN Gateway, Azure Bastion |
| DNS Resolver | Azure Private DNS Resolver with on-prem forwarding rules |
| Budget | 80 / 100 / 120 % forecast alerts |
| Profile | `platform-connectivity` |

### Identity (`platform-identity`)

Active Directory Domain Services with three deployment modes.

| Attribute | Value |
|-----------|-------|
| Deploy Order | **3** вҖ” after connectivity |
| Management Group | Platform |
| Identity Modes | `adds` В· `aadds` В· `hybrid` (recommended) |
| Key Resources | Domain Controller VMs (HA across 2 AZs), PAWs, Key Vault (AD secrets), PIM |
| NSG Rules | AD DS ports вҖ” 389, 636, 88, 445, 3268, 3269 |
| Budget | 80 / 100 / 120 % forecast alerts |
| Profile | `platform-identity` |

| Mode | When to Use |
|------|------------|
| `adds` | New Azure-only environment with no existing AD |
| `aadds` | Small deployments, no need for GPO or schema extensions |
| `hybrid` | Existing on-prem AD вҖ” extend with replica DCs **(recommended)** |

:::caution
Domain Controller VMs require static private IPs for DNS resolution. Ensure these are reserved in the hub VNet address plan **before** deploying identity.
:::

### Security (`platform-security`)

Dedicated security operations subscription вҖ” enforces **separation of duties** between SecOps and platform operations.

| Attribute | Value |
|-----------|-------|
| Deploy Order | **4** вҖ” after identity |
| Management Group | Platform |
| Subscription Purpose | Dedicated security operations (separation of duties from Ops) |
| Sentinel | вң… Dedicated workspace, 730-day retention, 10 data connectors, Fusion ML |
| Defender for Cloud | All 12 plans enabled, auto-provisioning, MDE integration |
| SOAR Playbooks | Block-IP, Isolate-VM, Revoke-Session, Enrich-ThreatIntel |
| Threat Intelligence | Microsoft TI + STIX/TAXII (prod), 3 watchlists |
| Key Vault | Private endpoint only, purge protection, RBAC |
| Compliance | CIS 2.0, ASB v3, NIST 800-53, ISO 27001, SOC 2, PCI DSS (prod) |
| Budget | 80 / 100 / 120 % forecast alerts |
| Profile | `platform-security` |

#### SOAR Playbook Approval Gates

| Playbook | Auto in Dev? | Auto in Prod? | Rationale |
|----------|-------------|--------------|-----------|
| Block-SuspiciousIP | Simulation only | вң… Auto | Low user impact, fast containment |
| Isolate-CompromisedVM | Simulation only | вқҢ Human approval | High impact вҖ” takes VM offline |
| Revoke-EntraIDSession | Simulation only | вқҢ Human approval | Direct user impact |
| Enrich-ThreatIntel | вң… Auto | вң… Auto | Read-only enrichment |

:::danger[Non-negotiable]
The security subscription must **never** share a Sentinel workspace with the management subscription in production. Use `workspace_mode: dedicated` for full separation of duties and independent retention policies.
:::
