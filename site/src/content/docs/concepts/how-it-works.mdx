## Philosophy

**AI Orchestrates В· Humans Decide В· Azure Executes**

The accelerator uses specialized AI agents that collaborate through a structured
multi-step workflow. Each agent has a defined responsibility, produces specific
artifacts, and hands off to the next agent in the pipeline.

### Per-LZ Agents (Steps 1-7)

Execute the full APEX workflow for a single landing zone вҖ” from requirements
gathering through deployment and documentation.

### Estate Agents

Operate across all landing zones for cross-cutting governance, policy
enforcement, and RBAC management.

### Day-2 Agents (Steps 8-9)

Continuous operations вҖ” compliance monitoring, drift detection, and
auto-remediation with rollback capability.

## Key Principles

1. **Human-in-the-Loop** вҖ” 6 approval gates where humans review and approve
2. **AVM-First** вҖ” Azure Verified Modules for standardization
3. **Security by Default** вҖ” 6 non-negotiable security rules enforced everywhere
4. **Cost Governance** вҖ” Budget alerts mandatory on every deployment
5. **No Local Azure CLI** вҖ” All Azure operations through GitHub Actions
6. **Prefix-Based Naming** вҖ” Everything parameterized for multi-tenant reuse

## Artifact Flow

Each workflow step produces a numbered artifact that feeds into the next step:

```
01-requirements.md вҶ’ 02-architecture-assessment.md вҶ’ 03-design-*.md
    вҶ’ 04-governance-constraints.md вҶ’ 04-implementation-plan.md
    вҶ’ infra/{bicep,terraform}/ вҶ’ 06-deployment-summary.md вҶ’ 07-*.md
```

All artifacts are stored in `agent-output/{project}/` with estate-level
state tracked in `agent-output/00-estate-state.json`.
