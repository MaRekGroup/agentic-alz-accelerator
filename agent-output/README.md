# Agent Output — Per-Customer Generated Content

This directory holds **all customer-specific artifacts** generated by the accelerator agents.
Each customer/engagement gets its own subdirectory:

```
agent-output/
├── README.md               ← You are here (tracked in template)
├── contoso/                ← Customer engagement folder
│   ├── 00-estate-state.json
│   ├── tdd/                ← Technical Design Documents
│   ├── diagrams/           ← Architecture diagrams (.drawio, .mmd, .png)
│   ├── architecture/       ← Current/target state docs
│   ├── adr/                ← Architecture Decision Records
│   ├── assessment/         ← Brownfield assessment output
│   └── deliverables/       ← Generated .xlsx, .docx, .pptx
```

## Setup

Run `scripts/init-project.sh` to create a new customer folder:

```bash
./scripts/init-project.sh --customer acme --prefix acm --region eastus2
```

Or manually:

```bash
mkdir -p agent-output/{customer}/{tdd,diagrams,architecture,adr,assessment,deliverables}
cp environments/subscriptions.sample.json environments/subscriptions.json
# Edit environments/subscriptions.json with real values
```

## Gitignore

Customer folders (`agent-output/*/`) are **gitignored by default** so customer data
never leaks into the template repo. Only this README is tracked.

To track a specific customer's output (e.g., for a private fork):

```gitignore
# In .gitignore, add an exception:
!agent-output/my-customer/
```
